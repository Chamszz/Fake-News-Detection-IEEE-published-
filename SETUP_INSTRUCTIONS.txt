=====================================
WHAT THE SCRIPT DOES
=====================================

The script automatically:

1. Organizes your files into logical groups:
   - Setup files (requirements.txt, .gitignore)
   - Documentation (README.md)
   - Literature and methodology
   - Data preparation code
   - NLP preprocessing
   - BERT implementation
   - XGBoost implementation
   - Training and evaluation code
   - Results and analysis

2. Creates commits across 21 days (going back from today)
   - Each day gets 1-3 commits
   - Commits are spread randomly between 9 AM - 11 PM
   - Commit messages match the file category

3. Creates a professional README.md
   - Includes project overview
   - Model architecture diagram
   - Performance metrics
   - Usage instructions
   - Project structure

4. Pushes everything to GitHub
   - Uses your GitHub token for authentication
   - Forces update (overwrites if repo has old data)
   - Sets correct author information

=====================================
TROUBLESHOOTING
=====================================

Problem: "Project folder not found"
Solution: Check that PROJECT_DIR path is correct in the script
          Default: C:\Users\heman\OneDrive\Desktop\Fake News Detection

Problem: "Push failed - Authentication failed"
Solution: Check your GitHub token is correct
          Make sure GitHub repository exists and is empty
          Test token: https://github.com/settings/tokens

Problem: "Nothing to commit"
Solution: This is normal - means all files were already committed
          The script continues with the next batch

Problem: "Files are too large"
Solution: Script automatically skips files over 90MB
          These are added to .gitignore
          You can manually exclude large files by editing .gitignore

Problem: Script freezes or takes too long
Solution: Close and restart
          Check internet connection
          Ensure GitHub is not blocking the push

=====================================
CUSTOMIZATION
=====================================

You can customize the script by editing these values:

DAYS = 21              → Number of days to spread commits over
COMMITS_PER_DAY = (1, 3)  → Min and max commits per day
START_DAYS_AGO = 21    → How far back to start dating commits
WORK_HOURS = (9, 23)   → When commits appear to be made

Example: Make commits appear to be made over 14 days
DAYS = 14
START_DAYS_AGO = 14
